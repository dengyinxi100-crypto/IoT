## 一、设计目的

随着我国人口老龄化进程加速，"银发经济"和智慧养老已成为国家科技战略的重要方向。物联网技术作为智慧养老的核心支撑，能够为老年人居家生活提供安全监护、健康管理、生活便利等多方面的保障。

本次课程设计以**银发智慧居家**为应用场景，基于ESP32-S3微控制器和物联网网关技术，设计并实现一套面向老年人居家生活的智能监护系统。主要目的包括：

1. **知识目标**：深入理解物联网四层架构（感知层、网络层、平台层、应用层），掌握ESP-IDF开发框架、FreeRTOS实时操作系统、WebSocket实时通信技术及多种传感器驱动开发方法。

2. **能力目标**：独立完成从需求分析、硬件选型、嵌入式软件开发到Web前端可视化的全栈开发流程；掌握I2C、UART、ADC、PWM等外设接口编程；具备传感器数据采集、融合处理与实时推送的能力。

3. **素养目标**：培养工程实践能力、系统化思维和创新意识，树立"科技服务民生"的理念，理解物联网技术在智慧养老领域的实际应用价值。

## 二、设计内容与要求

本项目基于ESP-IDF v5.1.6开发框架，采用FreeRTOS多任务架构，围绕"银发智慧居家"场景实现以下核心功能：

### （一）基础功能

| 序号 | 功能模块 | 功能描述 |
|------|----------|----------|
| 1 | 温湿度采集 | AHT20传感器通过I2C接口读取环境温湿度，周期200ms |
| 2 | 光照检测 | 光敏传感器ADC采集环境光照强度，用于智能照明判断 |
| 3 | 燃气检测 | MQ-2传感器ADC采集可燃气体浓度，四级分级告警 |
| 4 | 人体存在检测 | 24GHz毫米波雷达（LD2410），IO电平+UART数据帧双通道融合检测 |
| 5 | WiFi通信 | ESP32-S3 STA模式连接家庭路由器，断线自动重连 |
| 6 | Web仪表盘 | HTTP Server（端口80）提供自适应移动端Web界面 |
| 7 | 实时数据推送 | WebSocket Server（端口81）200ms周期推送传感器JSON数据 |

### （二）银发场景扩展功能（实现4项）

| 编号 | 功能名称 | 实现描述 |
|------|----------|----------|
| 1 | 智能照明控制 | 自动模式（暗+有人→开白光）+手动开/关 + 手动5分钟无人自动恢复 |
| 2 | 环境异常告警 | 燃气中度以上触发蜂鸣器+RGB红灯闪烁；温度>60℃蜂鸣器告警 |
| 3 | 一键呼救按钮 | 板载BOOT按键（GPIO0），硬件消抖300ms，按下触发/再次按下解除SOS |
| 4 | 人体感应照明 | 24GHz毫米波雷达检测有人且环境暗时自动开灯，无人时延时3秒关灯 |

### （三）LED状态指示设计

| 颜色 | 触发条件 | 含义 |
|------|----------|------|
| 绿色 | 燃气轻度 + 无人 | 轻微异常提示 |
| 黄色 | 燃气中度告警 | 中度告警 |
| 红色 | 燃气重度 / SOS紧急求助 | 严重告警/紧急 |
| 白色 | 自动开灯 / 手动开灯 | 照明开启 |
| 熄灭 | 正常待机 | 系统正常 |

## 三、系统整体架构设计

### 3.1 物联网四层架构

本系统遵循物联网典型四层架构设计：

```
┌─────────────────────────────────────────────────────────────┐
│                      应用层 (Application)                    │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐  │
│  │ Web仪表盘     │  │ 灯光控制      │  │ SOS一键求助       │  │
│  │ (HTTP:80)    │  │ (手动/自动)   │  │ (按键+Web触发)    │  │
│  └──────────────┘  └──────────────┘  └──────────────────┘  │
├─────────────────────────────────────────────────────────────┤
│                      平台层 (Platform)                       │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  ESP32-S3 网关: FreeRTOS 多任务调度 + 数据融合处理      │   │
│  │  WebSocket Server (81) 实时推送 + HTTP Server (80)    │   │
│  └──────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│                      网络层 (Network)                        │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  WiFi STA (2.4GHz) 连接家庭路由器                       │   │
│  │  协议: TCP/IP + HTTP + WebSocket                       │   │
│  │  断线自动重连机制 (10次重试, 间隔5s)                     │   │
│  └──────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│                      感知层 (Perception)                     │
│  ┌────────┐ ┌────────┐ ┌──────┐ ┌────────┐ ┌────────────┐ │
│  │ AHT20  │ │ 光敏    │ │ MQ-2 │ │ 毫米波  │ │ SOS按钮     │ │
│  │ 温湿度 │ │ 传感器  │ │ 燃气 │ │ 雷达    │ │ (GPIO0)    │ │
│  └────────┘ └────────┘ └──────┘ └────────┘ └────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 各层在银发居家场景中的作用

- **感知层**：部署在老人居住环境中，实时采集温湿度、光照、燃气浓度、人体存在等环境数据，以及SOS按钮的用户主动求助信号。这是系统的"五官"，为上层决策提供原始数据。

- **网络层**：ESP32-S3通过WiFi STA模式接入家庭路由器，利用TCP/IP协议栈提供HTTP和WebSocket双通道通信能力。实现传感器数据上行和控制指令下行的双向数据传输。

- **平台层**：运行在ESP32-S3上的FreeRTOS多任务系统作为本地智能网关，承担数据融合（雷达IO+UART双通道融合）、逻辑决策（燃气分级、自动照明规则）、实时推送（WebSocket广播）等核心处理任务。

- **应用层**：Web仪表盘作为主要用户界面，支持手机/平板浏览器访问，提供实时环境数据展示、灯光远程控制、SOS一键求助等交互功能。

### 3.3 系统数据流向

```
传感器数据: 传感器 → I2C/ADC/UART驱动 → sensor_task(200ms周期采集)
           → 数据融合处理(LED控制/告警判定) → WebSocket广播 → Web仪表盘实时更新

控制指令: Web仪表盘 → HTTP GET /cmd → cmd_handler解析
        → ON/OFF/AUTO/SOS → LED/蜂鸣器执行 → WebSocket广播状态同步

SOS触发:  硬件按键(GPIO0) → sos_task(300ms消抖) → SOS状态切换
        → RGB红灯+蜂鸣器60s + WebSocket广播全端通知
```

## 四、智能设备硬件架构设计

### 4.1 主控芯片选型分析

本系统选择**ESP32-S3**作为主控芯片，基于以下多维度对比分析：

| 对比维度 | ESP32-S3 | STM32F4 | Arduino Mega | 树莓派4B |
|----------|----------|---------|--------------|----------|
| 内核 | Xtensa LX7 双核 240MHz | Cortex-M4 180MHz | ATmega2560 16MHz | Cortex-A72 四核 1.5GHz |
| WiFi/BLE | 内置 2.4G WiFi + BLE5 | 需外挂模块 | 需外挂模块 | 内置 |
| RAM | 512KB SRAM + 8MB PSRAM | 192KB SRAM | 8KB SRAM | 2-8GB |
| GPIO | 45个可编程 | 最多140个 | 54个 | 26个 |
| ADC | 2×12-bit SAR ADC | 3×12-bit ADC | 10-bit ADC | 无板载ADC |
| 开发框架 | ESP-IDF (FreeRTOS) | HAL/STM32Cube | Arduino IDE | Linux/Raspbian |
| 功耗 | ~200mA (WiFi) | ~100mA | ~50mA | ~3A |
| 价格 | ¥25-40 | ¥30-60 | ¥80-120 | ¥300-500 |

**选型理由**：

1. **内置WiFi/BLE**：ESP32-S3原生支持2.4GHz WiFi和BLE5.0，无需额外通信模块，大幅简化硬件设计和降低BOM成本。STM32和Arduino均需外挂WiFi模块（如ESP-01），增加硬件复杂度和成本。

2. **双核FreeRTOS**：Xtensa LX7双核处理器配合FreeRTOS实现真正的多任务并行，传感器采集、WebSocket通信、HTTP服务可分配在不同任务中运行，互不干扰。相比Arduino的单线程loop架构具有本质优势。

3. **丰富外设接口**：2组I2C、3组UART、2组12-bit SAR ADC、8路LEDC PWM，完全满足本项目所有传感器（AHT20-I2C、光敏-ADC、MQ-2-ADC、雷达-UART+IO、RGB-PWM）的接口需求。

4. **成熟的开发生态**：ESP-IDF v5.1.6提供完整的驱动库、网络协议栈（lwIP）、HTTP Server、mbedTLS等组件，加速开发进程。

5. **性价比最优**：单价¥25-40，相比树莓派4B（¥300+）大幅降低成本；相比STM32+外挂WiFi方案（¥60+）也更经济。

### 4.2 传感器与执行器选型

| 器件 | 型号 | 选型理由 | 接口 | 参数 |
|------|------|----------|------|------|
| 温湿度传感器 | AHT20 | 高精度(±0.3℃/±2%RH)，I2C接口节省GPIO，比DHT11精度更高响应更快 | I2C(0x38) | 0.1℃/0.1%RH分辨率 |
| 光敏传感器 | 光敏电阻模块 | 模拟+数字双输出，性价比高，满足室内光照检测需求 | ADC1_CH0 + GPIO2 | 12-bit ADC |
| 气体传感器 | MQ-2 | 对液化气、甲烷、烟雾灵敏度高，适合家庭燃气泄漏检测 | ADC1_CH2 + GPIO4 | 12-bit ADC |
| 毫米波雷达 | LD2410(24GHz) | 可检测静止人体（区别于普通PIR仅检测移动），穿透力强 | UART2 + GPIO5 | 115200bps |
| RGB LED | 共阳极模块 | 三色独立PWM控制，通过颜色指示不同告警等级 | 3×LEDC PWM | 13-bit分辨率 |
| 蜂鸣器 | 有源蜂鸣器 | 驱动简单，低电平触发，适合告警提示 | GPIO21 | 3.3V |
| SOS按钮 | BOOT按键 | 利用板载按键，无需额外硬件，一键触发/解除 | GPIO0 | 上拉输入 |

## 五、智能设备硬件原理图设计

本项目使用面包板+杜邦线进行原型搭建，未进行PCB设计。各模块接线方式如下：

### 5.1 AHT20 温湿度传感器（I2C）

| AHT20引脚 | ESP32-S3引脚 | 说明 |
|-----------|-------------|------|
| VCC | 3.3V | 供电 |
| GND | GND | 地 |
| SDA | GPIO41 | I2C数据线，需4.7kΩ上拉 |
| SCL | GPIO42 | I2C时钟线，需4.7kΩ上拉 |

### 5.2 光敏传感器（ADC + GPIO）

| 模块引脚 | ESP32-S3引脚 | 说明 |
|----------|-------------|------|
| VCC | 3.3V | 供电 |
| GND | GND | 地 |
| AO | GPIO1 | 模拟输出→ADC1_CH0，光照越暗ADC值越高 |
| DO | GPIO2 | 数字输出，低电平=亮（阈值可调电位器） |

### 5.3 MQ-2 气体传感器（ADC + GPIO）

| 模块引脚 | ESP32-S3引脚 | 说明 |
|----------|-------------|------|
| VCC | 5V | 需5V供电（加热丝） |
| GND | GND | 地 |
| AO | GPIO3 | 模拟输出→ADC1_CH2，浓度越高ADC值越高 |
| DO | GPIO4 | 数字输出（阈值可调） |

### 5.4 毫米波雷达 LD2410（UART + IO）

| 雷达引脚 | ESP32-S3引脚 | 说明 |
|----------|-------------|------|
| VCC | 5V | 供电 |
| GND | GND | 地 |
| IO | GPIO5 | 存在检测输出，低电平=有人 |
| RX | GPIO6(TX) | UART2，ESP接收雷达数据 |
| TX | GPIO7(RX) | UART2，ESP发送配置命令 |

### 5.5 RGB LED（PWM）

| 模块引脚 | ESP32-S3引脚 | 说明 |
|----------|-------------|------|
| VCC | 3.3V | 共阳极供电 |
| R | GPIO40 | 红色通道 LEDC_CH0 |
| G | GPIO39 | 绿色通道 LEDC_CH1 |
| B | GPIO38 | 蓝色通道 LEDC_CH2 |

**注意**：共阳极LED需反相驱动——PWM占空比100%=灭，0%=最亮。

### 5.6 蜂鸣器与SOS按钮

| 器件 | 引脚 | ESP32-S3 | 说明 |
|------|------|----------|------|
| 有源蜂鸣器 | VCC | GPIO21 | 低电平触发鸣响，高电平静音 |
| BOOT按键 | - | GPIO0 | 板载按键，按下为低电平（上拉输入） |

### 6.3 LED控制优先级决策流程

```
sensor_task(200ms) → 采集所有传感器数据
                   → led_control(气体等级, 光敏ADC, 雷达有人)
                   → 决策优先级:
                      1. SOS激活?  → 红色 (最高优先级)
                      2. 手动模式? → 跳过自动控制
                      3. 燃气≥重度  → 红色
                      4. 燃气=中度  → 黄色
                      5. 暗+有人    → 白色 (自动开灯)
                      6. 燃气=轻度+无人 → 绿色
                      7. 其他       → 熄灭
```

### 6.4 WebSocket 通信流程

```
客户端(手机浏览器)                     ESP32-S3 网关
      │                                      │
      │──── WebSocket 连接 ws://IP:81 ──────→│ TCP accept + HTTP握手
      │                                      │ SHA1(Key+GUID)→Accept
      │←── 101 Switching Protocols ──────────│ WebSocket连接建立
      │                                      │
      │←── JSON传感器数据 (每200ms) ────────│ ws_broadcast()
      │   {temperature,humidity,             │ 所有已连接客户端
      │    light_adc,gas_adc,gas_level,      │
      │    radar,wifi,heap,manual,sos}       │
      │                                      │
      │── HTTP GET /cmd?c=ON ───────────────→│ cmd_handler()
      │←── "OK" ────────────────────────────│ LED + 广播状态同步
```

## 七、智能设备软件设计代码实现

### 7.1 主程序入口 (app_main)

主程序负责系统初始化、WiFi连接等待、服务和任务启动：

```c
void app_main(void)
{
    // 1. NVS 初始化（WiFi凭证存储）
    nvs_flash_init();

    // 2. 同步对象创建
    g_evt_group = xEventGroupCreate();    // WiFi/SOS事件标志
    g_sensor_mutex = xSemaphoreCreateMutex(); // 传感器数据互斥
    ws_mutex = xSemaphoreCreateMutex();      // WebSocket客户端列表互斥

    // 3. 硬件初始化（6个外设驱动）
    hardware_init();   // RGB LED → ADC → AHT20 → 光敏 → MQ-2 → BSP → 雷达

    // 4. WiFi STA 启动
    wifi_init_sta();

    // 5. 等待WiFi连接（15秒超时）
    xEventGroupWaitBits(g_evt_group, EVT_WIFI_CONNECTED, ...);

    // 6. 启动HTTP + WebSocket服务 + 3个FreeRTOS任务
    start_http_server();
    xTaskCreate(ws_server_task, "ws_server", 4096, ...);  // 优先级+3
    xTaskCreate(sensor_task,    "sensor",    4096, ...);  // 优先级+2
    xTaskCreate(sos_task,       "sos_btn",   2048, ...);  // 优先级+3
}
```

**设计要点**：使用EventGroup协调WiFi连接状态，Mutex保护共享数据（传感器读数、WebSocket客户端列表），任务栈大小和优先级根据实际负载设定。

### 7.2 传感器采集与智能控制 (sensor_task)

这是系统的核心任务，每200ms执行一次完整的采集-判断-执行-广播循环：

```c
static void sensor_task(void *arg)
{
    while (1) {
        // 1. 采集全部传感器
        aht20_read(&t, &h);                    // I2C温湿度
        light_sensor_read_adc(&light_adc);      // ADC光照
        mq2_read_adc(&gas_adc);                // ADC燃气
        int radar_pre = radar_presence_read();  // GPIO存在检测
        radar_read_uart_data();                 // UART数据帧(非阻塞)
        int radar_combined = (radar_pre || radar_uart) ? 1 : 0; // 双通道融合

        // 2. LED控制（仅在状态变化时更新）
        if (条件变化) led_control(gas_lev, light_adc, radar_combined);

        // 3. 手动开灯后5分钟无人 → 自动转AUTO关灯
        if (g_manual_mode == 1 && !radar_combined) { /* 超时判断 */ }

        // 4. 蜂鸣器告警（燃气中度以上 或 温度>60℃）
        if (gas_lev >= 2 || t > 60.0f) buzzer_alarm(3000);

        // 5. WebSocket广播JSON数据
        ws_broadcast(json, jlen);

        vTaskDelay(pdMS_TO_TICKS(200));
    }
}
```

**设计要点**：
1. 双通道雷达融合：GPIO5（500ms响应延迟）+ UART数据帧（精确目标类型），任一通道有效即判定有人，兼顾实时性与准确性。
2. `led_control()`采用优先级决策链，SOS>燃气>智能照明>待机，确保紧急情况优先响应。
3. 仅在状态变化时调用LED更新，减少不必要的PWM操作。

### 7.3 SOS按键任务（硬件消抖 + 切换逻辑）

```c
#define SOS_DEBOUNCE_MS  300  // 消抖时间300ms

static void sos_task(void *arg)
{
    int stable_state = 1;  // 默认未按下（上拉输入）
    while (1) {
        int raw = sos_button_pressed() ? 0 : 1;

        // 状态变化 → 启动消抖计时
        if (raw != last_state) { debounce_start = xTaskGetTickCount(); }

        // 消抖时间到达 → 确认状态变化
        if (消抖完成 && raw != stable_state) {
            stable_state = raw;
            if (stable_state == 0) {  // 确认按下
                if (g_sos_active) {
                    // 再次按下 → 解除SOS
                    g_sos_active = 0;  rgb_led_off();  buzzer_alarm(0);
                    ws_broadcast("{\"sos\":0,\"event\":\"sos_clear\"}");
                } else {
                    // 首次按下 → 触发SOS
                    g_sos_active = 1;  rgb_led_set_color(RED);
                    buzzer_alarm(60000);  // 持续60秒蜂鸣
                    ws_broadcast("{\"sos\":1,\"event\":\"sos_alert\"}");
                }
            }
        }
        vTaskDelay(pdMS_TO_TICKS(50));  // 50ms轮询周期
    }
}
```

**设计要点**：
1. GPIO0是ESP32-S3的strapping引脚，Boot ROM已配置为上拉输入，驱动层不重新配置避免影响芯片启动。
2. 300ms消抖窗口兼顾响应速度与机械抖动过滤。
3. **切换逻辑**：同一按键首次按下触发SOS，再次按下解除，符合老年人简单操作的需求。
4. SOS状态通过WebSocket实时广播到所有Web客户端，实现全端同步。

### 7.4 AHT20 温湿度传感器驱动

```c
esp_err_t aht20_read(float *temperature, float *humidity)
{
    // 1. 发送触发测量命令 0xAC 0x33 0x00
    uint8_t trigger[3] = {0xAC, 0x33, 0x00};
    i2c_master_write_to_device(I2C_NUM_0, 0x38, trigger, 3, ...);

    vTaskDelay(pdMS_TO_TICKS(80));  // 等待80ms测量完成

    // 2. 读取6字节数据
    uint8_t data[6];
    i2c_master_read_from_device(I2C_NUM_0, 0x38, data, 6, ...);

    // 3. 解析20-bit原始值 → 物理量
    //    湿度(%) = raw_humi / 2^20 * 100
    //    温度(℃) = raw_temp / 2^20 * 200 - 50
    uint32_t raw_humi = (data[1]<<12) | (data[2]<<4) | (data[3]>>4);
    uint32_t raw_temp = ((data[3]&0x0F)<<16) | (data[4]<<8) | data[5];

    *humidity    = (float)raw_humi * 100.0f / 1048576.0f;
    *temperature = (float)raw_temp * 200.0f / 1048576.0f - 50.0f;
}
```

### 7.5 毫米波雷达LD2410协议解析

```c
int radar_read_uart_data(void)
{
    uint8_t buf[128];
    int len = uart_read_bytes(UART_NUM_2, buf, 128, 0);  // 非阻塞读取

    // 在字节流中搜索LD2410数据帧: FD FC FB FA ... 04 03 02 01
    for (int i = 0; i + 13 <= len; i++) {
        if (buf[i]!=0xFD || buf[i+1]!=0xFC || buf[i+2]!=0xFB || buf[i+3]!=0xFA)
            continue;

        int data_len = buf[i+4] | (buf[i+5] << 8);

        // 验证帧尾 04 03 02 01
        if (buf[frame_end] != 0x04 || ...) continue;

        // 读取目标状态 (偏移8): 0=无人, 1=运动, 2=静止, 3=运动+静止
        uint8_t target_state = buf[i + 8];
        g_radar_uart_presence = (target_state != 0x00) ? 1 : 0;
    }
}
```

**设计要点**：LD2410毫米波雷达能检测静止人体（区别于传统PIR红外传感器），特别适合老年人在房间内静坐/睡眠等场景的准确检测。

## 八、系统调试与功能验证

### 8.1 调试过程

1. **CMake路径问题**：项目从 `E:/物联网网关设计` 迁移到 `E:/IoT` 后，CMakeCache.txt记录的是旧路径，导致 `CMake Error: source directory mismatch`。解决方法：删除build目录后重新执行 `cmake -B build -S .`。

2. **ESP-IDF扩展配置**：VS Code中ESP-IDF扩展未正确识别项目目录，原因是工作区根目录 `E:\IoT` 不含CMakeLists.txt（实际在 `E:\IoT\IotGateway`）。解决方法：在 `.vscode\settings.json` 中添加 `"idf.projectDir": "E:/IoT/IotGateway"`。

3. **雷达检测问题**：初始仅使用GPIO5 IO引脚检测，发现更新频率低（约500ms）。优化为IO引脚+UART数据帧双通道融合检测，UART帧可提供更高频率的目标状态（含静止/运动区分），两者互补提高检测准确性和实时性。

4. **共阳极LED驱动**：RGB LED为共阳极模块，初始代码未做反相处理导致颜色异常。解决方法：在 `rgb_led_set_rgb()` 中增加反相逻辑 `duty = max_duty - duty`。

### 9.2 实用性分析

1. **银发居家安全监护**：燃气泄漏检测+蜂鸣器告警可有效防止老人因嗅觉退化或健忘导致的燃气事故；SOS一键求助为紧急情况（摔倒、突发疾病）提供快速求助通道。

2. **智能照明辅助**：夜间有人自动开灯解决老人起夜时摸黑找开关的痛点，白光照明可在5米范围内提供清晰视野；手动5分钟无人自动关灯避免能源浪费。

3. **环境舒适度监控**：温湿度实时监测可间接反映老人居住环境健康状况，异常高温自动蜂鸣提醒降温或通风。

4. **低学习成本**：Web仪表盘自适应手机屏幕，界面简洁（深色主题，大字体），仅4个按钮（开灯/关灯/自动/SOS），适合老年人操作。

### 9.3 可维护性与升级路径

- **模块化代码结构**：传感器驱动独立封装，添加新传感器只需新增驱动文件并注册。
- **配置集中管理**：所有引脚、阈值、WiFi凭证集中在 `app_config.h`，修改配置无需改动业务逻辑。
- **OTA升级**（预留）：ESP32-S3支持通过WiFi OTA固件升级，可在不拆机情况下远程更新。
- **云平台接入**：可在WebSocket基础上增加MQTT客户端，接入阿里云IoT/EMQX等平台，实现远程监控和历史数据存储。

### 9.4 创新点

1. **毫米波雷达替代传统PIR**：LD2410 24GHz雷达可检测静止人体，解决PIR传感器"人静止就误判为无人"的痛点，显著提升人体存在检测准确率。双通道（IO+UART）融合进一步提高可靠性。

2. **SOS按钮切换机制**：单按键实现"触发/解除"双向控制，避免需要独立解除按钮的复杂度，降低老人操作混淆风险。

3. **WebSocket全栈实时通信**：200ms周期推送全部传感器数据至Web前端，延迟低、无轮询开销，相比传统HTTP轮询更高效。

4. **LED多级语义指示**：RGB LED通过6种颜色/模式表达不同系统状态（正常/轻度/中度/重度/照明/SOS），实现"一目了然"的无声状态感知。

通过本次"基于ESP32的银发智慧居家场景全栈应用设计与开发"课程设计，我获得了以下收获与思考：

**1. 物联网全栈开发的完整认知**。从传感器驱动的底层I2C/ADC/UART编程，到FreeRTOS多任务调度的中间件设计，再到WebSocket实时通信和Web前端可视化，我亲身体验了物联网系统从底层到顶层的完整开发流程，深刻理解了四层架构中每一层的职责和实现方式。

**2. 硬件驱动开发的工程实践**。编写AHT20、MQ-2、LD2410等传感器的驱动代码时，需要仔细阅读芯片数据手册（Datasheet），理解I2C通信时序、ADC采样配置、UART协议帧解析等底层细节。例如AHT20的20-bit原始值解析公式、LD2410的FD FC FB FA帧格式，都需要精确按照协议实现。

**5. 科技服务民生**。本次设计面向银发居家场景，让我认识到技术应该以人为本——简化操作（单键SOS）、降低学习成本（Web界面自适应手机）、关注真实需求（夜间照明、燃气安全），这些设计理念将指导我未来的工程实践。
