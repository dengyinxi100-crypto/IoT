## 一、项目背景与目标

### 1.1 行业背景

近年来，全球人口老龄化趋势持续加剧。根据国家统计局发布的第七次全国人口普查数据，我国60岁及以上人口已达2.64亿，占总人口的18.7%，预计到2035年这一比例将突破30%。在"9073"养老格局下（90%居家养老、7%社区养老、3%机构养老），绝大多数老年人选择居家养老模式。

然而，居家养老面临诸多现实挑战：老年人嗅觉功能退化导致燃气泄漏不能及时发现、夜间行动不便容易跌倒、突发疾病时缺乏及时求助手段等。传统的智能家居方案存在部署成本高（单套网关+传感器套装通常500-2000元）、操作复杂（需要APP注册、配网、配置场景）、依赖云端（断网即失效）等问题，在普通家庭特别是老年人群体中普及率较低。

物联网技术的快速发展为解决上述问题提供了全新路径。以ESP32-S3为代表的国产高性价比物联网芯片日趋成熟，24GHz毫米波雷达传感器已经实现低成本民用化，WebSocket实时通信协议为本地局域网低延迟数据交互提供了标准方案。基于此技术背景，本项目旨在设计一套**低成本、离线可用、操作极简**的银发智慧居家监护系统。

### 1.2 项目目标

本项目以"银发智慧居家"为应用场景，基于ESP32-S3微控制器和ESP-IDF开发框架，设计并实现一套完整的物联网智能家居网关原型系统。项目目标涵盖以下四个层面：

**（1）安全监护。** 实现可燃气体泄漏的实时检测与分级告警，提供一键紧急求助（SOS）功能，为独居老人构建基本的安全防线。

**（2）生活辅助。** 实现基于毫米波雷达和光敏传感器的智能照明——夜间老人起身时自动亮灯，避免摸黑行动导致的跌倒意外。

**（3）远程感知。** 家属或看护人可通过手机浏览器实时查看老人的居住环境数据（温湿度、光照、燃气浓度、人员活动状态），无需安装任何APP。

**（4）低门槛部署。** 硬件总成本控制在100元以内，操作仅需一个按键和手机浏览器，WiFi断线不影响本地核心功能运行。

### 3.2 数据流设计

系统数据流分为上行采集和下行控制两个方向。

**上行数据流（传感器→用户）：**

```
传感器硬件 ──→ 驱动层 ──→ sensor_task(200ms周期) ──→ JSON封装 ──→ WebSocket全客户端广播
              I2C/ADC/                                (512B JSON)
              UART/GPIO                  │
                                  ┌──────┴──────┐
                                  ▼              ▼
                            LED控制决策      蜂鸣器告警判断
                           (优先级链)       (燃气/温度)
```

**下行数据流（用户→执行器）：**

```
Web页面按钮 ──→ HTTP GET /cmd?c=ON ──→ cmd_handler ──→ 全局状态修改
                                                        │
                                          ┌─────────────┼─────────────┐
                                          ▼             ▼             ▼
                                       LED状态      手动模式      WebSocket
                                       立即响应      计时器       全端同步广播
```

**SOS特殊流程（双向触发）：**

```
物理按键按下 ──→ sos_task消抖(300ms) ──→ 确认有效
                                          │
                              ┌───────────┴───────────┐
                              ▼                       ▼
                       g_sos_active切换         WebSocket广播
                     (按=触发/再按=解除)          "sos_alert"/"sos_clear"
                              │
                    ┌─────────┼─────────┐
                    ▼         ▼         ▼
                RGB红灯    蜂鸣器60s   Web页面红条闪烁
```

## 四、硬件架构设计

### 4.1 主控芯片选型

网关主控芯片的选择直接决定了系统的性能边界、外设扩展能力和总体物料成本。本项目从处理性能、通信能力、外设资源、开发生态和单价五个维度，对当前主流的四款嵌入式平台进行了横向对比。

**表4-1 主控芯片对比分析**

| 对比维度 | ESP32-S3 | STM32F407 | Arduino Mega | 树莓派4B |
|:--------:|----------|-----------|:------------:|:--------:|
| CPU | Xtensa LX7 双核240MHz | Cortex-M4 168MHz | ATmega2560 16MHz | Cortex-A72 四核1.5GHz |
| 无线通信 | **内置**WiFi+BLE5 | 无（需外挂模块） | 无（需外挂模块） | 内置WiFi+BLE |
| SRAM | 512KB+8MB PSRAM | 192KB | 8KB | 2-8GB |
| GPIO | 45个 | ≤140个 | 54个 | 26个 |
| ADC | 2×12-bit SAR | 3×12-bit | 10-bit | 无 |
| I2C / UART | 2组 / 3组 | 3组 / 4组 | 1组 / 4组 | 1组 / 1组 |
| PWM | 8路LEDC(13-bit) | 多路定时器 | 15路(8-bit) | 2路 |
| RTOS | FreeRTOS（内置） | 支持移植 | 不支持 | 重量级OS |
| 单价(元) | 25–40 | 30–60 | 80–120 | 300–500 |

**选型结论：ESP32-S3**

- **WiFi集成**是刚性需求——网关必须具备无线通信能力，STM32和Arduino方案需外挂WiFi模块额外增加15-20元硬件成本和AT指令开发复杂度，树莓派的功耗和成本远超需求。
- **双核+FreeRTOS**为多任务并行提供了原生支持——传感器采集、WebSocket通信、HTTP服务可在不同任务中并发运行。
- **外设资源充足**——本项目使用2路ADC、1路I2C、1路UART、3路PWM、4路GPIO，ESP32-S3资源留有2倍以上余量。
- **单价25-40元**，在满足全部功能需求的前提下硬件成本最低。

## 五、硬件原理图设计

本项目使用面包板和杜邦线完成原型搭建。以下逐一说明各模块的电路连接细节。

### 5.1 AHT20 温湿度传感器（I2C）

**表5-1 AHT20接线表**

| 模块引脚 | ESP32-S3 | 说明 |
|:--------:|:--------:|------|
| VCC | 3.3V | 供电电压范围2.0-5.5V |
| GND | GND | 共地 |
| SDA | GPIO41 | I2C数据线，建议外接4.7kΩ上拉电阻 |
| SCL | GPIO42 | I2C时钟线，建议外接4.7kΩ上拉电阻 |

I2C通信速率100kHz（标准模式）。每次测量流程：发送触发命令（0xAC 0x33 0x00）→等待80ms→读取6字节原始数据→20-bit非标准位宽解析→转换为物理量。上电后需先发送初始化命令（0xBE）并等待40ms完成内部校准。

### 5.2 光敏电阻传感器（ADC + 数字IO）

**表5-2 光敏传感器接线表**

| 模块引脚 | ESP32-S3 | 说明 |
|:--------:|:--------:|------|
| VCC | 3.3V | 供电 |
| GND | GND | 共地 |
| AO | GPIO1 | 模拟输出→ADC1_CH0，**光照越暗→ADC值越高**（反比关系） |
| DO | GPIO2 | 数字输出，低于模块电位器阈值时输出低电平 |

自动照明判定阈值设为 `AUTO_LIGHT_THRESHOLD = 2000`，即 ADC>2000 视为"暗环境"。

### 5.3 MQ-2 气体传感器（ADC + 数字IO）

**表5-3 MQ-2传感器接线表**

| 模块引脚 | ESP32-S3 | 说明 |
|:--------:|:--------:|------|
| VCC | **5V** | **必须5V供电**——加热丝需5V达到工作温度（~300℃），3.3V无法正常工作 |
| GND | GND | 共地 |
| AO | GPIO3 | 模拟输出→ADC1_CH2，浓度越高→ADC值越高 |
| DO | GPIO4 | 数字输出，浓度超过电位器阈值时输出低电平 |

上电后需预热20-60秒才能获得稳定读数。浓度分级标准：0-800正常、800-1500轻度、1500-2500中度、2500+重度/危险。

### 5.4 LD2410 毫米波雷达（UART + 数字IO）

**表5-4 LD2410雷达接线表**

| 雷达引脚 | ESP32-S3 | 说明 |
|:--------:|:--------:|------|
| VCC | 5V | 功耗约0.5W |
| GND | GND | 共地 |
| IO | GPIO5 | 存在检测输出，低电平=有人，高电平=无人 |
| RX | GPIO6(TX) | **交叉连接**：MCU的UART2_TX接雷达RX |
| TX | GPIO7(RX) | **交叉连接**：MCU的UART2_RX接雷达TX |

UART参数：115200bps, 8N1。采用**双通道融合策略**：GPIO5 IO口简单可靠（约500ms更新），UART数据帧更新高频（约100ms）且含目标运动/静止细分信息，两者取逻辑或作为最终判据。

### 5.5 RGB LED（3路PWM）

**表5-5 RGB LED接线表**

| 模块引脚 | ESP32-S3 | LEDC配置 | 说明 |
|:--------:|:--------:|:--------:|------|
| VCC | 3.3V | — | 共阳极公共端 |
| R | GPIO40 | LEDC_CH0 | 13-bit分辨率, 5kHz |
| G | GPIO39 | LEDC_CH1 | 同上 |
| B | GPIO38 | LEDC_CH2 | 同上 |

**共阳极驱动原理：** PWM占空比100%（高电平）→LED两端电压差为0→熄灭；占空比0%（低电平）→电位差最大→最亮。代码中通过 `duty = max_duty - duty` 反相处理。

### 5.6 蜂鸣器与SOS按键

**表5-6 蜂鸣器与SOS按键接线表**

| 器件 | 连接 | ESP32-S3 | 说明 |
|:----:|:----:|:--------:|------|
| 有源蜂鸣器 | VCC | GPIO21 | 低电平蜂鸣，高电平静音 |
| SOS按键 | — | GPIO0 | 板载BOOT键，按下=低电平（GND连接），松开=高电平 |

GPIO0为ESP32-S3的Strapping引脚，芯片上电时ROM Bootloader自动配置为上拉输入，驱动代码**不**对此引脚进行重新配置，直接使用 `gpio_get_level(GPIO_NUM_0)` 读取。

### 6.4 WebSocket 通信设计

WebSocket协议用于传感器数据的实时推送，解决传统HTTP轮询的延迟和带宽浪费问题。

- **握手流程**：客户端发起HTTP Upgrade请求（端口81），服务端提取`Sec-WebSocket-Key`，拼接GUID魔数`258EAFA5-E914-47DA-95CA-C5AB0DC85B11`，SHA-1哈希+Base64编码生成`Sec-WebSocket-Accept`，返回HTTP 101 Switching Protocols。
- **数据帧封装**：首字节0x81（FIN=1+文本帧），payload长度自适应编码（<126单字节，<65536三字节，≥65536九字节），通过`sendmsg()`+`iovec`散列-聚集I/O原子发送帧头与payload。
- **广播策略**：sensor_task每200ms构造JSON（约250-350字节），遍历已连接客户端列表（最多4个）逐一发送。

### 7.2 传感器采集与智能控制任务

```c
static void sensor_task(void *arg)
{
    char json[512];
    int last_gas = 0xff, last_light = -1, last_radar = -1;

    while (1) {
        /* 采集全部传感器 */
        float t = 0, h = 0; int light_adc, gas_adc;
        aht20_read(&t, &h);
        light_sensor_read_adc(&light_adc);
        mq2_read_adc(&gas_adc);
        int radar_io = radar_presence_read();
        radar_read_uart_data();
        int radar = (radar_io || radar_presence_uart_get()) ? 1 : 0;
        int gas_lev = get_gas_level(gas_adc);

        /* LED控制：仅在状态变化时更新 */
        if (gas_lev != last_gas || light_adc != last_light
            || radar != last_radar) {
            led_control(gas_lev, light_adc, radar);
            last_gas = gas_lev; last_light = light_adc;
            last_radar = radar;
        }

        /* 手动模式无人超时恢复 */
        if (g_manual_mode == 1 && !radar) {
            if ((xTaskGetTickCount()-g_manual_on_ticks)
                >= pdMS_TO_TICKS(MANUAL_ON_TIMEOUT_MS)) {
                g_manual_mode = 0; rgb_led_off();
            }
        }

        /* 蜂鸣器告警 */
        if (gas_lev >= 2 || t > 60.0f) buzzer_alarm(3000);

        /* WebSocket广播 */
        int jlen = snprintf(json, sizeof(json),
            "{\"id\":%d,\"temperature\":%.1f,\"humidity\":%.1f,"
            "\"light_adc\":%d,\"gas_adc\":%d,\"gas_level\":%d,"
            "\"radar\":%d,\"wifi\":%d,\"heap\":%d,"
            "\"manual\":%d,\"sos\":%d}",
            seq++, t, h, light_adc, gas_adc, gas_lev,
            radar, wifi_ok, (int)esp_get_free_heap_size(),
            g_manual_mode, g_sos_active);
        ws_broadcast(json, jlen);

        vTaskDelay(pdMS_TO_TICKS(200));
    }
}
```

**要点：**
- 状态变化检测（`gas_lev != last_gas || ...`）避免每个周期都重新配置PWM。
- 手动模式5分钟无人超时自动关灯并切回自动模式，兼顾用户主动操作与节能。
- 蜂鸣器覆盖燃气安全和高温安全两个告警维度。
- JSON帧约250-350字节，局域网延迟<5ms。

### 7.3 SOS按键任务

```c
#define SOS_DEBOUNCE_MS  300

static void sos_task(void *arg)
{
    int stable_state = 1;
    while (1) {
        int raw = sos_button_pressed() ? 0 : 1;
        if (raw != last_state) {
            debounce_start = xTaskGetTickCount();
            debouncing = 1; last_state = raw;
        }
        if (debouncing &&
            (xTaskGetTickCount()-debounce_start)
                >= pdMS_TO_TICKS(SOS_DEBOUNCE_MS)) {
            debouncing = 0;
            if (raw != stable_state) {
                stable_state = raw;
                if (stable_state == 0) {  /* 确认按下 */
                    if (g_sos_active) {
                        /* 已有SOS → 解除 */
                        g_sos_active = 0;
                        rgb_led_off(); buzzer_alarm(0);
                        ws_broadcast("{\"sos\":0,\"event\":\"sos_clear\"}");
                    } else {
                        /* 无SOS → 触发 */
                        g_sos_active = 1;
                        rgb_led_set_color(RGB_COLOR_RED);
                        buzzer_alarm(60000);
                        ws_broadcast("{\"sos\":1,\"event\":\"sos_alert\"}");
                    }
                }
            }
        }
        vTaskDelay(pdMS_TO_TICKS(50));
    }
}
```

**要点：**
- 经典状态机消抖：电平跳变启动300ms定时器，到期后电平仍保持则确认为有效按键。
- **乒乓切换**：同一按键按一次触发，再按一次解除。通过`g_sos_active`标志实现状态记忆。
- 每次SOS状态变更立即WebSocket广播全端同步。
- 蜂鸣器60秒自动关闭使用FreeRTOS one-shot软件定时器（不阻塞任务）。

### 7.4 AHT20 温湿度传感器驱动

```c
esp_err_t aht20_read(float *temperature, float *humidity)
{
    /* 触发测量：0xAC 0x33 0x00 */
    uint8_t cmd[3] = {0xAC, 0x33, 0x00};
    i2c_master_write_to_device(I2C_NUM_0, 0x38, cmd, 3, 50);
    vTaskDelay(pdMS_TO_TICKS(80));

    aht20_wait_ready();  /* 等待忙标志清除 */

    /* 读取6字节原始数据 */
    uint8_t data[6] = {0};
    i2c_master_read_from_device(I2C_NUM_0, 0x38, data, 6, 50);

    /* 解析20-bit非标准编码
     * 湿度: Byte[1]全8-bit + Byte[2]全8-bit + Byte[3]高4-bit
     * 温度: Byte[3]低4-bit + Byte[4]全8-bit + Byte[5]全8-bit */
    uint32_t raw_humi = ((uint32_t)data[1]<<12) |
                        ((uint32_t)data[2]<<4)  |
                        ((uint32_t)data[3]>>4);
    uint32_t raw_temp = (((uint32_t)data[3]&0x0F)<<16) |
                        ((uint32_t)data[4]<<8) | data[5];

    /* 转换为物理量（2^20=1048576） */
    *humidity    = (float)raw_humi * 100.0f / 1048576.0f;
    *temperature = (float)raw_temp * 200.0f / 1048576.0f - 50.0f;

    return ESP_OK;
}
```

### 7.5 LD2410 雷达UART协议解析

```c
int radar_read_uart_data(void)
{
    uint8_t buf[128];
    int len = uart_read_bytes(UART_NUM_2, buf, 128, 0);  /* 非阻塞 */
    if (len <= 0) return -1;

    for (int i = 0; i + 13 <= len; i++) {
        /* 匹配帧头：FD FC FB FA */
        if (buf[i]!=0xFD || buf[i+1]!=0xFC ||
            buf[i+2]!=0xFB || buf[i+3]!=0xFA) continue;

        int data_len = buf[i+4] | (buf[i+5] << 8);
        int frame_end = i + 6 + data_len;
        if (frame_end + 4 > len) break;

        /* 验证帧尾：04 03 02 01 */
        if (buf[frame_end]!=0x04 || buf[frame_end+1]!=0x03 ||
            buf[frame_end+2]!=0x02 || buf[frame_end+3]!=0x01) continue;

        /* 目标状态(偏移8)：0x00=无人 0x01=运动 0x02=静止 0x03=运动+静止 */
        uint8_t target_state = buf[i + 8];
        g_radar_uart_presence = (target_state != 0x00) ? 1 : 0;
        return len;
    }
    return len;
}
```

**要点：** 帧头（FD FC FB FA）和帧尾（04 03 02 01）的双重校验确保协议解析可靠性。目标状态非0x00即判定有人，不依赖具体的运动/静止分类。

### 8.1 开发环境适配

**（1）CMake缓存冲突。** 项目从原路径迁移后编译报错 `source directory mismatch`。根因：`build/CMakeCache.txt` 记录了旧路径的绝对地址。解决：删除build目录重新执行 `cmake -B build -S .`。

**（2）IDE扩展配置。** VS Code未识别项目目录。根因：工作区根目录不含CMakeLists.txt（实际子在目录中）。解决：`.vscode/settings.json` 添加 `"idf.projectDir": "E:/IoT/IotGateway"`。

### 8.2 硬件调试

**（1）雷达检测延迟。** GPIO5 IO引脚状态更新约500ms，人员进出有明显延迟。优化方案：IO+UART双通道融合——UART帧更新约100ms且含运动/静止细分信息。

**（2）RGB LED颜色反转。** 期望红色但显示青色。根因：模块为共阳极结构，PWM需反相驱动。解决：`rgb_led_set_rgb()`中增加 `duty = max_duty - duty` 反相逻辑。

**（3）MQ-2预热。** 上电前30秒ADC读数从2500持续下降至400。方案：将传感器采集任务的启动放在WiFi连接等待之后，自然覆盖预热时间段。

- **云平台接入：** 网络层已为MQTT接入做好准备，后续可对接阿里云IoT/EMQX/OneNET，实现远程管理、历史数据存储和数据看板。

### 9.4 创新点

**（1）毫米波雷达替代传统PIR红外。** LD2410 24GHz雷达可以检测**静止**人体（如静坐、睡眠状态），解决了PIR传感器（HC-SR501等）"人不动就误判无人"的痛点。IO+UART双通道融合（IO简单可靠、UART高频细粒度）进一步确保了检测覆盖的完整性。这一技术选型契合银发居家场景中老人长时间静止的实际生活状态。

**（2）SOS单键乒乓式切换。** 仅需一个按键（板载BOOT键），按一次触发紧急求助，再按一次解除。无需独立的"取消"按钮，降低老人操作混淆风险。每次状态变更通过WebSocket广播所有已连接的Web客户端，家属可即时获知。

**（3）WebSocket全栈实时通信。** 传感器数据（温度、湿度、光照、燃气、雷达、内存）以200ms周期通过WebSocket推送到所有Web客户端。在局域网环境下延迟<5ms，无需轮询、实时性好。相比传统的MQTT+云端转发方案，省去了云端依赖和网络穿透成本。

**（4）RGB LED多级语义指示。** 一个RGB LED灯珠承载6种系统状态信息：绿色（轻微异常提示）、黄色（中度告警）、红色（重度告警/SOS）、白色（照明开启）、熄灭（正常待机），实现"无需看屏幕、一眼知状态"的直觉式信息表达。

## 十、项目总结

本项目围绕"银发智慧居家"应用场景，基于ESP32-S3微控制器和ESP-IDF v5.1.6开发框架，设计并实现了一套完整的物联网智能家居网关原型系统。

**完成的主要工作：**

1. 完成了从需求分析、架构设计、硬件选型与搭建、嵌入式软件开发到功能验证的全流程。

2. 实现了10项核心功能：温湿度采集、光照检测、燃气分级监测、毫米波雷达人体存在检测、RGB LED多级状态指示、蜂鸣器告警、智能照明控制（自动+手动）、SOS一键求助、Web监控面板、WebSocket实时数据推送。

3. 实现了4项银发场景扩展功能：智能照明远程控制、环境异常分级告警、单键SOS紧急求助（乒乓切换）、人体感应智能照明（静止人体检测）。

4. 硬件BOM成本控制在90元以内，具备规模化生产的成本基础。

**存在的问题与改进方向：**

- 当前为面包板原型阶段，后续可进行PCB设计和小型化封装。
- 可增加MQTT客户端模块，实现云平台接入和远程管理。
- 可引入eFuse或安全启动机制，增强设备安全性。
- Web界面可增加历史数据曲线展示和告警记录回溯功能。
